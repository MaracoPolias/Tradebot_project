# Editor Guide
**tradebot-project**

This guide provides editorial standards for contributors responsible for reviewing and refining documentation within the repository.

---

## 1. Editorial Responsibilities
Editors are expected to:
- Improve clarity and readability  
- Ensure consistent formatting  
- Maintain professional tone  
- Correct grammar and sentence structure  
- Organize content logically  
- Standardize terminology across the project  
- Verify that examples and code blocks are accurate  

---

## 2. Writing Standards
- Use concise, clear, direct language  
- Avoid ambiguous or vague statements  
- Prefer active voice  
- Keep paragraphs short for readability  
- Avoid excessive technical jargon when simpler wording works  
- Ensure consistency in spelling (US English preferred)  

---

## 3. Markdown Formatting
- Use `#` headings correctly  
- Use proper fenced code blocks  
- Keep lines narrow (GitHub-friendly)  
- Use tables sparingly but effectively  
- Avoid raw HTML unless necessary  
- Ensure links work  

---

## 4. Documentation Review Workflow
1. Read content carefully  
2. Identify unclear or inconsistent areas  
3. Make edits directly or comment with suggestions  
4. Confirm code examples run correctly  
5. Verify terminology matches existing docs  
6. Submit PR in `docs/*` branch naming format:


docs/improve-<section-name>


---

## 5. Common Terms (Standardized)
- “strategy engine”  
- “risk manager”  
- “execution pipeline”  
- “signal generator”  
- “threshold tuning”  
- “module” (avoid “component” unless necessary)

---

## 6. Tone and Voice
- Professional  
- Calm  
- Neutral  
- Helpful  
- No unnecessary hype or overly casual phrasing  

---

Editors, please also ensure that tradebot-project remains polished and accessible.  
Thank you for your work and attention to detail.
